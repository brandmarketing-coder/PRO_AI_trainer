# 功能測試清單

每個功能的測試流程與確認項目。逐項打勾即完成驗收；交接沙龍部門前建議完整跑一輪，日後大改版後也可重跑。

- 測試網址：https://pro-ai-trainer.onrender.com/
- 密碼：主管＝`REPORT_PASSWORD`（預設 12890464）；管理員＝`ADMIN_PASSWORD`（Render 環境變數）
- 建議用手機與電腦各測一輪（介面為手機優先設計）

---

## 1. 首頁與全站

**測試流程**：開啟網站首頁 → 觀察版面 → 縮放視窗（或換手機）看排版。

- [ ] 頂欄顯示 O'right logo ＋「Jenny老師」，深綠底
- [ ] 首頁功能卡為線條圖示（無 emoji）、低圓角卡片
- [ ] 「指定演練」卡只在有開放題目時出現，右上顯示題數
- [ ] 手機（窄螢幕）無左右捲動、按鈕可正常點擊
- [ ] 後台設定公告後，公告以整條色帶顯示在頂欄正下方（見 7.2）

## 2. 情境演練

**測試流程**：首頁 → 情境演練 → 選主題（如陌生開發）→ 選模式（新人）→ 選姓名 → 對練 3～4 輪 → 右上「結束並評分」→ 看報告 → 下載 Word/PDF。

- [ ] 三步驟設定流程順暢，姓名可從名單選
- [ ] 店長回話符合角色（不自稱業務、不跳出角色）
- [ ] 每句話可展開「💡 看看怎麼說可以更好」
- [ ] **回饋針對「你剛說的那句」**：調整建議＝這句怎麼講更好；「這句可以這樣講」＝同一句的改寫版（不是幫你想下一句）
- [ ] 講出禁用話術（試講「這個可以治療掉髮」）會跳即時糾錯
- [ ] 評估報告含五大構面、六關卡、改善建議、總評
- [ ] Word 與 PDF 可下載開啟、格式正常
- [ ] 演練後「分數彙整」出現這筆紀錄（見 7.1），Google Sheet「演練紀錄」分頁多一列（見 9）

## 3. 知識問答

**測試流程**：首頁 → 知識問答 → 點一個建議問題 → 再手動問「咖啡因養髮液的容量和價格」→ 再問一個知識庫沒有的（如「你們有賣香水嗎」）。

- [ ] 起始畫面有分類建議問題，點了直接送出
- [ ] 產品題以「業務理解／產品重點／可以這樣說／提醒」結構回答，數字與知識庫一致
- [ ] 常見題（FAQ 快取）幾乎秒回
- [ ] 知識庫沒有的內容回「目前資料中沒有看到明確說明」，不亂編
- [ ] 回答為繁體中文、純文字排版（無 # * 等符號）

## 4. 隨機測驗

**測試流程**：首頁 → 隨機測驗 → 填姓名 → 選範圍 → 作答 2～3 題 → 結束測驗看成績。

- [ ] 範圍選擇前有姓名欄位（可從名單選）
- [ ] 出題與批改順暢，逐題有評語與參考答案
- [ ] 成績頁顯示答對率與逐題回顧，可下載測驗報告 Word
- [ ] Google Sheet「測驗成績」分頁多一列（時間/業務/範圍/題數/答對/正確率）

## 5. 指定演練（Jenny 的每週題目）

**測試流程（管理員）**：報表 → 指定演練分頁 → 出一題（填題目/說明/評分重點）→ 發布。
**測試流程（業務）**：回首頁 → 指定演練卡出現 → 進入做題 → 上傳音檔（或直接打字）→ 確認逐字稿 → 送出評分。
**測試流程（收錄）**：後台繳交紀錄 → 勾「優良」→ 匯出勾選 → 將勾選收錄進知識庫。

- [ ] 出題後**不用重整**，回首頁就看到指定演練卡與題數
- [ ] 上傳音檔自動轉逐字稿（1 分鐘音檔約 10～20 秒），可修正錯字
- [ ] 評分結果依「本題評分重點」逐項給 ◎○△＋點評，另有五大構面參考、總分、優缺點、總評
- [ ] 繳交後後台看得到（分數彙整頁也有「指定演練繳交」表）
- [ ] 勾「優良」後「匯出勾選的話術」下載文字檔，內容含姓名/分數/逐字稿
- [ ] 「將勾選收錄進知識庫」確認後顯示已收錄；知識庫管理分頁出現 `14_優良話術示範.md`
- [ ] 收錄後到知識問答問相關話術，AI 會參考優良示範
- [ ] 編輯題目、關閉開放、刪除題目皆正常；關閉後首頁卡即時消失

## 6. 權限（兩級密碼）

**測試流程**：分別用主管密碼與管理員密碼進報表。

- [ ] 主管密碼：只看得到「分數彙整」分頁
- [ ] 管理員密碼：看得到全部五個分頁
- [ ] 主管密碼直接打管理功能 API 會被拒絕（介面上看不到入口即可）
- [ ] 輸入錯誤密碼顯示「密碼錯誤」，且稽核日誌留下「登入失敗」
- [ ] **記住登入**：解鎖後回首頁再進報表，不用重新輸入密碼（同一分頁有效）
- [ ] 右上「登出」後回到密碼畫面；關閉瀏覽器分頁後重開也需重新輸入

## 7. 報表後台

### 7.1 分數彙整
- [ ] 表格含：業務／次數／最近練習時間／最近分數／層級／待加強面向
- [ ] 未練習者為灰字「尚未練習」（無紅色）
- [ ] 下方有「指定演練繳交（最近 20 筆）」表，含收錄狀態
- [ ] 電腦版表格寬度舒適（960px），不擠

### 7.2 系統管理（管理員）
- [ ] 關閉「隨機測驗」→ 回首頁該卡**立即**消失（不用重整）；業務直接呼叫也會被擋
- [ ] 重新開啟 → 首頁卡立即恢復
- [ ] 設定公告 → 首頁頂欄下方立即出現整條色帶；清空後立即消失
- [ ] 名單增刪一人並儲存 → 演練姓名選單立即反映
- [ ] 上述變更顯示「已永久保存到 GitHub」（約 2～4 分鐘後重新部署，設定不會丟）
- [ ] 稽核日誌記錄了以上所有動作（時間/動作/內容/身分）

### 7.3 知識庫管理（管理員）
- [ ] 檔案清單顯示儲存位置為 GitHub
- [ ] 「檢視」開啟大視窗，內容排版好讀、可捲動、點 X 或背景可關閉
- [ ] 上傳 .md 直接存檔；貼上雜亂文字由 AI 整理成 Markdown
- [ ] 上傳/刪除後提示「重新部署後套用」；稽核日誌有紀錄

### 7.4 資料備份（管理員）
- [ ] 顯示演練筆數、指定演練筆數、上次備份時間、備份位置（GitHub）
- [ ] 「即時歸檔」顯示 Google Sheet（Apps Script 直連）
- [ ] 「立即備份到 GitHub」成功並更新時間
- [ ] 「下載完整備份（JSON）」可下載，內容含 records/audit/submissions

## 8. 資料保存與還原（重要，交接前必測）

**測試流程**：完成一場演練 → 後台按「立即備份」→ 到 Render 按 Manual Deploy（模擬重新部署）→ 部署完成後看分數彙整。

- [ ] 重新部署後，演練紀錄自動從 GitHub 備份還原（分數彙整資料還在）
- [ ] repo 的 `backups/records.json` 有對應 commit（git 歷史＝備份歷史）

## 9. Google Sheet 歸檔

**測試流程**：分別完成一場情境演練、一次指定演練繳交、一次測驗 → 開 Google 試算表。

- [ ] 「演練紀錄」分頁多一列，時間為台北時區、逐字稿完整
- [ ] 「指定演練」分頁多一列，含重點評分/做得好/待加強/整體評語
- [ ] 「測驗成績」分頁多一列，含正確率與逐題摘要
- [ ] 分頁與標題列自動建立（若第一次寫入）

## 10. 環境變數總表（Render → Environment）

| 變數 | 用途 | 必要性 |
|---|---|---|
| `PROVIDER` / `OPENAI_API_KEY` | AI 對話與音檔轉寫 | 必填 |
| `REPORT_PASSWORD` | 主管密碼 | 建議改掉預設 |
| `ADMIN_PASSWORD` | 管理員密碼 | **交接前必設** |
| `GITHUB_TOKEN` | 知識庫寫回／資料備份 | 必填 |
| `APPS_SCRIPT_URL` | Google Sheet 即時歸檔 | 建議 |
| `TRANSCRIBE_MODEL` | 轉寫模型（預設 whisper-1） | 選填 |
| `N8N_WEBHOOK_URL` | 舊歸檔路徑（已由 APPS_SCRIPT_URL 取代） | 可刪 |
